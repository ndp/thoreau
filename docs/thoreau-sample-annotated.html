test_suite 'addition!' do subject do a + b end spec 'addition works', setup: 'a is 37', input: { b: 5 }, expect: 42 appendix do setup 'a is 37', { a: 32 + 4 + 1 } end end
test_suite wraps a set of specs Suites always start with the subject block, the "code under test". This helps make test suites easy to scan. Tests are largely declarative: a name, one or more setups, inputs, and expectations for output. Since most assertions are for equality, it's the default, but free-form assertions are available. The appendix is a place to relegate messy code, like the setup blocks. The appendix helps readers see the intent-- rather than implementation-- of the test suite and its tests.
